<decisions_made>

- Signal priority: XGBoost (primary) → MR (fallback) → Pairs (fallback) — first active source wins
- Source-specific exits: xgb → p <= sell_threshold; mr → RSI>55 OR bb_pos>0.6; pairs → spread revert/stop
- PairState created fresh inside run_backtest (not passed in) — each sweep run starts clean
- Pairs only uses BTC/ETH pair, only acts on sig.pair == "BTC/USD" signals
- BTC is laggard when zscore < 0 (BTC cheap relative to ETH)
- MR call: stateless single-row slice `feat_df.iloc[i:i+1]` — O(1), no warmup needed
- Pairs efficiency: `btc_idx.searchsorted(ts)` for O(log n) timestamp lookup + 500-bar cap

</decisions_made>
