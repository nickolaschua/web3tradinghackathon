**Tech stack available:** Python 3.11, systemd, AL2023, chrony (pre-configured), venv
**Key decisions from research:**
- AL2023: use `python3.11` explicitly — `python3` is 3.9, DO NOT reassign symlink
- chrony: Amazon Time Sync already pre-configured on AL2023 via `/etc/chrony.d/link-local.sources` — verification only, no config write needed
- venv location: inside repo at `/home/ec2-user/bot/venv` (simpler paths, already gitignored)
- systemd `EnvironmentFile=` injects vars before Python starts; `load_dotenv()` in main.py is harmless (systemd vars take precedence; leave it for local dev fallback)
- `ReadWritePaths=/home/ec2-user/bot` (whole dir) — avoids pitfall where `state.json` doesn't exist yet and systemd refuses to start
- Do NOT add `MemoryDenyWriteExecute=yes` — breaks Python JIT (numpy/pandas crash at import)
- Use `network-online.target` not `network.target` — ensures IP is assigned before bot connects to mock-api.roostoo.com
- `.env` format for systemd: bare `KEY=VALUE` — NO quotes, NO `export`, NO spaces around `=`
- `main.py` reads `ROOSTOO_API_KEY` and `ROOSTOO_SECRET` (not `_TEST` variants) — for smoke test, set testing key values under the non-TEST variable names in EC2 `.env`
</context>

Create the `deploy/` directory and write the systemd service unit file using the exact pattern from 08-RESEARCH.md code examples.

Unit must have:
- `[Unit]` section: Description, `After=network-online.target`, `Wants=network-online.target`
- `[Service]` section:
  - `Type=simple`
  - `User=ec2-user` and `Group=ec2-user`
  - `WorkingDirectory=/home/ec2-user/bot`
  - `EnvironmentFile=/home/ec2-user/bot/.env`
  - `Environment=PYTHONUNBUFFERED=1`
  - `ExecStart=/home/ec2-user/bot/venv/bin/python main.py` — absolute path to venv Python, NO `source activate`
  - `Restart=on-failure` (NOT `always` — masks rapid crash loops)
  - `RestartSec=10`
  - `StandardOutput=journal` and `StandardError=journal`
  - Hardening (safe subset for Python + pandas/numpy):
    - `NoNewPrivileges=yes`
    - `PrivateTmp=yes`
    - `ProtectSystem=strict`
    - `ReadWritePaths=/home/ec2-user/bot` — whole repo dir; covers state.json (which doesn't exist until first write), logs/, and any future files
    - `RestrictAddressFamilies=AF_INET AF_INET6`
    - `RestrictRealtime=yes`
    - `LockPersonality=yes`
    - **NO** `MemoryDenyWriteExecute=yes` — Python JIT (numpy/pandas) uses W+X memory; this would crash the process silently at import
    - **NO** `ProtectHome=yes` — WorkingDirectory is under /home/ec2-user; this would conflict
- `[Install]` section: `WantedBy=multi-user.target`

Add a comment header block explaining: systemd EnvironmentFile= injects .env before Python starts, so ROOSTOO_API_KEY etc. are available via os.environ — no load_dotenv() needed in production.

Write two scripts to `deploy/`:

**deploy/bootstrap.sh** — one-shot setup script, run once after first SSH to a fresh AL2023 instance:
```bash
#!/bin/bash
set -e  # exit on first error

REPO_URL="REPLACE_WITH_YOUR_REPO_URL"  # placeholder; user edits before running
BOT_DIR="/home/ec2-user/bot"
VENV_DIR="${BOT_DIR}/venv"

echo "=== [1/6] Updating system packages ==="
sudo dnf update -y

echo "=== [2/6] Installing Python 3.11 and git ==="
sudo dnf install python3.11 python3.11-pip git -y

echo "=== [3/6] Cloning repo ==="
cd /home/ec2-user
git clone "$REPO_URL" bot
cd bot

echo "=== [4/6] Creating Python 3.11 venv (explicitly python3.11, NOT python3 which is 3.9) ==="
python3.11 -m venv "$VENV_DIR"
"${VENV_DIR}/bin/pip" install --upgrade pip
"${VENV_DIR}/bin/pip" install -r requirements.txt

echo "=== [5/6] Creating .env from template ==="
cp .env.example .env
chmod 600 .env
echo ""
echo "ACTION REQUIRED: Edit .env with your actual keys:"
echo "  nano ${BOT_DIR}/.env"
echo "  Set ROOSTOO_API_KEY and ROOSTOO_SECRET to your TESTING keys for smoke test"
echo "  Format: bare KEY=VALUE (no quotes, no export prefix)"
echo ""

echo "=== [6/6] Installing systemd service ==="
sudo cp deploy/roostoo-bot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable roostoo-bot.service
echo "Service installed and enabled. Start with: sudo systemctl start roostoo-bot.service"
echo ""
echo "=== Bootstrap complete ==="
```

**deploy/deploy.sh** — update script for pushing code changes after initial deployment:
```bash
#!/bin/bash
# Run from repo root on LOCAL machine to push updates to EC2
# Usage: ./deploy/deploy.sh <ec2-ip>
set -e

EC2_IP="${1:-}"
if [ -z "$EC2_IP" ]; then
  echo "Usage: ./deploy/deploy.sh <ec2-instance-ip>"
  exit 1
fi

EC2_HOST="ec2-user@${EC2_IP}"
BOT_DIR="/home/ec2-user/bot"
VENV_DIR="${BOT_DIR}/venv"

echo "Deploying to ${EC2_HOST}..."
ssh "$EC2_HOST" "
  set -e
  cd ${BOT_DIR}
  git pull origin main
  ${VENV_DIR}/bin/pip install -r requirements.txt
  sudo systemctl restart roostoo-bot.service
  echo 'Waiting 3s for service to start...'
  sleep 3
  sudo systemctl status roostoo-bot.service --no-pager
"
echo "Deploy complete."
```

Make both scripts executable: `chmod +x deploy/bootstrap.sh deploy/deploy.sh`

Note in a comment at the top of bootstrap.sh: "Edit REPO_URL before running. On AL2023, `python3` is 3.9 — this script explicitly uses `python3.11` everywhere."
